# 名片查询项目使用说明书

本文档说明本项目的日常使用方式，重点是后续新增、修改名片信息，以及如何发布到 GitHub Pages 让手机端同步新版数据。

当前部署方式已经从 Cloudflare 上传 zip 改为 GitHub Pages：

```text
main 分支保存项目源码
        ↓
GitHub Actions 自动执行
        ↓
把 mobile_search 文件夹同步到 gh-pages 分支
        ↓
GitHub Pages 从 gh-pages 分支发布网站
```

线上访问地址：

```text
https://jianshenghao2023-creator.github.io/Namecard/
```

GitHub 仓库：

```text
https://github.com/jianshenghao2023-creator/Namecard
```

## 1. 项目是做什么的

这个项目把扫描名片整理成联系人数据，并生成一个手机可用的查询网页。

手机端网页的工作方式是：

1. 联网打开网页时，自动下载最新名片数据。
2. 下载后保存到手机本地。
3. 之后即使没有网络，也可以继续查询上一次同步过的数据。

当前手机网页目录是：

```text
E:\namecard\mobile_search
```

当前手机网页使用的数据文件是：

```text
E:\namecard\mobile_search\contacts-data.json
```

备用的手机导入数据文件是：

```text
E:\namecard\mobile_data\namecard_contacts_data.json
```

## 2. 重要文件说明

### 联系人源数据

手机查询数据从这个 CSV 生成：

```text
E:\namecard\namecard_contacts_enriched_v1.csv
```

如果只修改 Excel 文件而不更新 CSV，手机端数据不会变化，因为生成脚本读取的是 CSV。

对应的 Excel 文件是：

```text
E:\namecard\namecard_contacts_enriched_v1.xlsx
```

Excel 更适合人工查看和编辑；CSV 是生成手机数据时实际读取的文件。

### 手机网页文件

GitHub Pages 发布的是 `mobile_search` 文件夹里的网页文件：

```text
E:\namecard\mobile_search
```

里面最关键的文件包括：

```text
index.html
styles.css
app-online-sync-v2.js
sw.js
manifest.webmanifest
icon.svg
contacts-data.json
```

其中 `contacts-data.json` 就是手机联网时自动同步的数据。

### 数据生成脚本

每次修改联系人 CSV 后，运行：

```powershell
cd E:\namecard
.\build_mobile_search.ps1
```

这个脚本会重新生成：

```text
E:\namecard\mobile_search\contacts-data.json
E:\namecard\mobile_data\namecard_contacts_data.json
E:\namecard\mobile_search_update.zip
```

说明：

- `mobile_search\contacts-data.json` 是 GitHub Pages 实际发布的数据文件。
- `mobile_data\namecard_contacts_data.json` 是手动导入备用文件。
- `mobile_search_update.zip` 是旧 Cloudflare 流程留下的备用打包文件，当前 GitHub Pages 日常更新不需要上传这个 zip。

## 3. 当前部署结构

4. GitHub Pages 自动发布 `gh-pages` 分支内容。

GitHub Pages 如果需要手动设置，选择：

```text
Settings -> Pages -> Build and deployment -> Deploy from a branch
Branch: gh-pages
Folder: / (root)
```

## 4. 修改已有名片信息

适用于改姓名、公司、职位、电话、邮箱、备注、分类等信息。

### 操作步骤

1. 打开：

```text
E:\namecard\namecard_contacts_enriched_v1.csv
```

也可以先打开 Excel 文件：

```text
E:\namecard\namecard_contacts_enriched_v1.xlsx
```

2. 找到要修改的联系人所在行。

3. 修改需要调整的字段。

常用字段包括：

```text
person_name              英文名或原名
chinese_name             中文名
company                  名片上的公司名
company_normalized       统一后的公司名
title                    职位
country                  国家
city                     城市
email_primary            主要邮箱
email_secondary          备用邮箱
mobile                   手机
phone                    固定电话
website                  网站
address                  地址
notes                    备注
industry_tags            行业标签
primary_category_code    分类编号
primary_category         主分类
subcategory              子分类
business_role            业务角色
```

4. 保存为 CSV UTF-8。

如果是从 Excel 编辑的，保存时请确认最终覆盖的是：

```text
E:\namecard\namecard_contacts_enriched_v1.csv
```

5. 回到项目目录，重新生成手机数据：

```powershell
cd E:\namecard
.\build_mobile_search.ps1
```

6. 提交并推送网站数据：

```powershell
git add mobile_search/contacts-data.json
git commit -m "Update namecard data"
git push
```

7. 等 GitHub Actions 和 GitHub Pages 部署完成。

## 5. 新增一张名片

目前新增名片需要先人工把名片信息整理成一行联系人记录。也就是说，这个项目现在不是全自动 OCR 系统，新增名片时需要人工确认姓名、公司、电话、邮箱等字段。

### 操作步骤

1. 打开：

```text
E:\namecard\namecard_contacts_enriched_v1.csv
```

2. 在最后新增一行。

3. 给新联系人填写一个新的 `record_id`。

当前已有记录到 `NC-0221`，所以下一条可以从：

```text
NC-0222
```

开始。后续依次递增，例如 `NC-0223`、`NC-0224`。

4. 至少填写这些字段：

```text
record_id
review_status
confidence
person_name 或 chinese_name
company
company_normalized
title
country
city
email_primary
mobile
phone
website
notes
primary_category_code
primary_category
subcategory
business_role
```

5. 推荐默认状态：

```text
review_status = 待校对
confidence = high 或 medium
relationship_stage = 名片已扫描
```

6. 如果这张名片来自某个展会或特殊来源，把来源写进 `notes`。

例如来自希腊海事展：

```text
2026希腊海事展
```

7. 如果有原始 PDF 或图片来源，也可以填写：

```text
source_pdf
source_page
card_position
source_image
```

这些字段现在手机页面不一定全部显示，但保留在数据里方便以后追溯。

8. 保存 CSV UTF-8。

9. 重新生成手机数据：

```powershell
cd E:\namecard
.\build_mobile_search.ps1
```

10. 提交并推送网站数据：

```powershell
git add mobile_search/contacts-data.json
git commit -m "Add new namecard"
git push
```

11. 等 GitHub Actions 和 GitHub Pages 部署完成。

## 8. 如果 GitHub Actions 没有自动发布

一般不需要手工处理。如果 Actions 没有自动运行，可以在 GitHub 网页执行：

```text
Actions -> Publish static site to gh-pages -> Run workflow
```

如果 workflow 异常，也可以在本地手动把 `mobile_search` 发布到 `gh-pages`：

```powershell
cd E:\namecard
git subtree split --prefix mobile_search -b gh-pages-deploy
git push --force origin gh-pages-deploy:gh-pages
git branch -D gh-pages-deploy
```

手动发布后，GitHub Pages 会从 `gh-pages` 分支重新构建。

## 9. 手机上如何同步新版数据

GitHub Pages 部署完成后：

1. iPhone 联网。
2. 打开“名片查询”网页或主屏幕图标。
3. 等待页面显示联系人数量。
4. 新版数据会自动下载并保存到手机本地。
5. 之后断网也可以继续查询。

如果手机仍然显示旧数据，可以按顺序尝试：

1. 刷新网页。
2. 退出主屏幕 App 后重新打开。
3. 等 30 秒再打开一次。
4. 如果仍不更新，进入 Safari 打开原始网址刷新一次。

## 10. 检查更新是否成功

电脑端检查：

1. 确认这个文件更新时间已经变化：

```text
E:\namecard\mobile_search\contacts-data.json
```

2. 确认脚本输出里的 `Contacts` 数量符合预期。

3. 确认线上数据文件已经更新：

```text
https://jianshenghao2023-creator.github.io/Namecard/contacts-data.json
```

手机端检查：

1. 打开名片查询。
2. 搜索新增加的人名或公司名。
3. 能查到就说明同步成功。

如果只是修改电话或邮箱，搜索对应联系人，查看详情是否已经变化。

## 11. 常见问题

### 为什么我修改了 Excel，但手机没有变化？

因为手机数据是从 CSV 生成的。请确认最终更新了：

```text
E:\namecard\namecard_contacts_enriched_v1.csv
```

然后重新运行：

```powershell
.\build_mobile_search.ps1
```

再提交并推送：

```powershell
git add mobile_search/contacts-data.json
git commit -m "Update namecard data"
git push
```

### 为什么手机上还是旧页面？

可能是浏览器缓存或离线缓存还没有更新。先联网刷新页面，再重新打开主屏幕图标。

### 还需要上传 mobile_search_update.zip 吗？

日常不需要。当前使用 GitHub Pages，实际发布的是 `gh-pages` 分支里的静态文件。

`mobile_search_update.zip` 只是旧 Cloudflare 部署流程的备用产物。除非以后临时恢复 Cloudflare 手动部署，否则不用上传这个 zip。

### 可以只提交 contacts-data.json 吗？

如果只修改了联系人数据，可以只提交：

```text
mobile_search/contacts-data.json
```

如果修改了网页样式、脚本、图标或 manifest，则提交整个：

```text
mobile_search
```

### 手机会不会每次都下载很多数据？

同一个浏览器会保存本地数据。联网打开时会检查并同步在线数据。当前数据只有几百 KB，体积很小。

不同浏览器的数据彼此隔离。Safari、Chrome、主屏幕 Web App 可能各自保存一份。建议固定用 Safari 添加到主屏幕后的图标。
